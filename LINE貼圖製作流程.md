## 四、Gemini 生圖規格

比例設定為 1:1，解析度使用 2K。
格式：3欄 × 3列，共 9 格
背景：每個格子的背景使用純綠色 #00FF00，不要有漸層或陰影
格線：格子之間有清楚的白色分隔線
角色外圍加上白色描邊（約 5-10px 粗細）
每個格子包含：角色動作 + 中文文字
文字在角色上方
文字使用白色描邊輪廓
角色和文字必須完全在格子範圍內，不能超出格線
角色距離格線至少留 10% 空白邊距
中文字正確度請加強

### 40 張套組需三張原稿：

| 原稿 | 格數 | 對應貼圖 |
|------|------|----------|
| 第一張（XX1-1） | 18 格 | 01–16 貼圖，第 17 格 = main，第 18 格 = tab |
| 第二張（XX1-2） | 18 格 | 17–32 貼圖（裝飾用格可留空） |
| 第三張（XX1-3） | 18 格 | 33–40 貼圖（裝飾用格可留空） |

> 注意：Gemini 生圖時偶爾順序會與預期不同，收到圖後先目視確認順序再跑腳本。

## 五、裁切注意事項（避免邊邊問題）

Gemini 原稿格線會造成相鄰格殘留，以下是各方向的處理原則：

| 方向 | 處理方式 |
|------|----------|
| 頂部（row > 0） | inset 8px，避免上一列底部格線滲入 |
| 左側（col > 0） | inset 2～3px，若仍有殘留可加大至 12～18px |
| 右側 | inset 2px，若文字接近右邊界則減少至 1px |
| 底部 | inset 2px |
| row = 0（第一列） | 頂部不加 inset（從格子最頂開始裁） |

### 常見問題與對策：

| 問題 | 原因 | 解法 |
|------|------|------|
| 頂部有細線 | 上一列底部格線 | 頂部 inset 加大至 8～12px |
| 左側有鄰格內容殘留 | 左邊格子圖案溢出 | 左側 inset 加大至 12～18px |
| 文字被截斷 | 右側 inset 太大 | 右側 inset 縮小至 1～2px |
| 主文字消失只剩副標 | autocrop 裁掉了文字上方空白 | 不做 autocrop，直接縮放 |
| 角色大小不一致 | 各格留白不同 | 統一縮放至 350×300 後置中於 370×320 畫布 |

## 六、自動化腳本使用方式

腳本位置：`F:/UserData/Desktop/my project/line auto pic/make_stickers.py`

### 使用步驟：

1. 將 Gemini 原稿放入 `gemini/` 資料夾
2. 在腳本頂部設定參數（套組名稱、原稿檔名、張數）
3. 執行腳本
4. 完成圖輸出至 `完成圖檔區/套組名稱/`
5. ZIP 自動輸出至 `zip儲存區/套組名稱.zip`

## 七、上傳流程

1. 前往 LINE Creator Market：https://creator.line.me/zh-hant/
2. 新增貼圖組 → 填入標題、說明（需中英文各一）
3. 上傳圖片 → 選「上傳 ZIP」→ 選取 `zip儲存區/套組名稱.zip`
4. LINE 自動辨識 main.png / tab.png 歸位
5. 提交審核（約 1～7 個工作天）
6. 審核通過後設定售價上架（最低 30 LINE Points ≈ NT$30）

## 十、N8N 自動化生產線

### 架構
```
N8N 表單 → Prompt 組合 → OpenRouter API 生圖（附參考圖）
  → 人工審核 → 本機 Webhook → make_stickers.py → ZIP
```

### 使用方式
1. 在 N8N 匯入 `line_sticker_generate_n8n.json`
2. 設定 OpenRouter API Key（HTTP Header Auth credential）
3. 本機啟動 webhook listener：`python local_webhook.py`
4. 搭配 ngrok 開放外網：`ngrok http 5000`
5. 打開 N8N 表單 → 填入角色資訊 → 送出即開始生圖

### 生圖 API（OpenRouter）
- 模型：`google/gemini-3.1-flash-image-preview`（flash）或 `google/gemini-3-pro-image-preview`（pro）
- 每次生圖帶參考圖（確保角色風格一致）
- 格式：1:1、2K、3×3 綠幕格子
- 每張 Sheet 間隔 15 秒避免 rate limit

### make_stickers.py CLI 參數
```bash
# 手動模式（沿用原本 CONFIG）
python make_stickers.py

# CLI 模式（N8N / webhook 自動呼叫）
python make_stickers.py --name otter_office --total 40 \
  --sheets "sheet1.png,sheet2.png,sheet3.png,sheet4.png,sheet5.png,sheet6.png" \
  --main 5,0 --tab 5,1
```

### 成本估算（每套 40 張）
- Flash 模型：約 **$0.27/套**（6 次 API 呼叫）
- Pro 模型：約 **$0.80/套**
- 生圖時間：約 2.5 分鐘
